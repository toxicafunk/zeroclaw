zeroclaw

El asistente de IA más rápido y pequeño.

Uso: zeroclaw [OPCIONES] <COMANDO>

Subcomandos:

  • onboard — Inicializa tu espacio de trabajo y configuración
  • agent — Iniciar el bucle del agente de IA
  • gateway — Iniciar/gestionar el servidor de gateway (webhooks, websockets)
  • acp — Iniciar el servidor ACP (Protocolo de Control de Agentes) a través de stdio
  • daemon — Iniciar el tiempo de ejecución autónomo de larga duración (gateway + canales + latido + programador)
  • service — Administrar el ciclo de vida del servicio del sistema operativo (servicio de usuario de launchd/systemd)
  • doctor — Ejecuta diagnósticos para la frescura del daemon/programador/canal
  • status — Mostrar el estado del sistema (detalles completos)
  • estop — Activar, inspeccionar y reanudar los estados de parada de emergencia
  • cron — Configurar y gestionar tareas programadas
  • models — Manage model_provider model catalogs
  • providers — List supported AI model_providers
  • channel — Administrar canales (Telegram, Discord, Slack)
  • integrations — Explora más de 50 integraciones
  • skills — Administrar habilidades (capacidades definidas por el usuario)
  • browse — Browse the shared workspace one directory at a time
  • sop — Gestionar procedimientos operativos estándar (SOP)
  • migrate — Migrar datos desde otros entornos de ejecución de agentes
  • auth — Manage model_provider subscription authentication profiles
  • hardware — Descubrir e inspeccionar el hardware USB
  • peripheral — Gestiona periféricos de hardware (STM32, GPIO de RPi, etc.)
  • memory — Administrar la memoria del agente (listar, obtener, estadísticas, limpiar)
  • config — Administrar la configuración
  • update — Buscar y aplicar actualizaciones
  • self-test — Ejecutar pruebas de diagnóstico
  • completions — Genera el script de autocompletado de shell en stdout
  • desktop — Iniciar o instalar la aplicación de escritorio complementaria

Opciones:

  • --config-dir <DIRECTORIO_CONFIG>

zeroclaw onboard

Inicializa tu espacio de trabajo y configuración

Usage: zeroclaw onboard [OPTIONS] [COMMAND]

Subcomandos:

  • providers.models — Pick a model provider to configure (Anthropic, OpenAI, OpenRouter, Ollama, custom OpenAI-compatible gateways, etc.). Multiple aliases per provider are supported — e.g. anthropic.production and anthropic.dev can coexist
  • risk-profiles — Named risk profiles binding allowlists, denylists, and approval thresholds. Agents reference one via agents.<alias>.risk_profile
  • runtime-profiles — Named runtime tuning profiles (token limits, retry policy, timeouts). Agents reference one via agents.<alias>.runtime_profile
  • storage — SQLite is the safe default for single-node installs (file-based, zero-config, no extra services). Pick Postgres for shared or multi-instance deployments, Qdrant for vector search, Markdown or Lucid for human-readable files. Each backend supports multiple aliased instances; agents reference them via memory.storage_ref
  • memory — Persistent memory backend. SQLite is the default; pick none to disable long-term recall entirely
  • skills — Skills tool settings — where skill markdown lives on disk (defaults to the data dir), and how the skills loader handles community repositories. Add skill BUNDLES under skill-bundles below
  • skill-bundles — Named bundles of skill files. Agents reference a bundle to load a set of capabilities at startup
  • mcp — Model Context Protocol settings. Toggle enabled and pick deferred or eager loading. Individual MCP servers live under mcp.servers[]
  • mcp-bundles — Named bundles of MCP servers. Agents reference a bundle to pull in a set of MCP tools as one unit
  • knowledge-bundles — Named bundles of knowledge sources (RAG indexes, doc folders). Agents reference a bundle to surface relevant snippets at inference time
  • providers.tts — Text-to-speech providers (OpenAI, ElevenLabs, Google, Edge, Piper). Configure one per voice / language; agents reference them by alias
  • providers.transcription — Speech-to-text providers (OpenAI Whisper, Groq, Deepgram, AssemblyAI, Google, local Whisper). Configure one per pipeline; agents reference them by alias
  • channels — Pick which chat platforms ZeroClaw should listen on. You can configure multiple — each channel gets its own alias
  • hardware — Optional: hardware peripherals (Arduino, STM32, GPIO, etc.). Skip if you don’t need them
  • agents — An agent binds a model provider, profiles, bundles, and channels into one dispatchable unit. Add one per persona; reuse the same alias across channels to share state
  • peer-groups — Named groups binding a channel, member agents, and external peers. Mutual opt-in: two agents become peers only when both appear in the same group’s agents list
  • cron — Scheduled tasks. Each cron entry binds a schedule expression to a prompt, channel, and target
  • tunnel — Optional: expose your gateway over the public internet via Cloudflare or ngrok. Pick none to keep it localhost-only

Opciones:

  • --quick — Skip interactive prompts; read from –api-key/–model-provider/–model/–memory
  • --cli — Force the dialoguer CLI backend instead of the default ratatui TUI
  • --force — Don’t ask “keep stored secret?” — always re-prompt
  • --reinit — Back up existing config and start from defaults
  • --api-key <API_KEY> — API key for model_provider configuration
  • --model-provider <MODEL_PROVIDER> — ModelProvider name. Used as the type key for the synthesized [model_providers.<type>.default] entry
  • --model <MODEL> — Model ID override
  • --memory <MEMORY> — Memory backend (sqlite, lucid, markdown, none)

zeroclaw agent

Iniciar el bucle del agente de IA.

Launches an interactive chat session with the configured AI model_provider. Use –message for single-shot queries without entering interactive mode.

Examples: zeroclaw agent -a assistant # interactive session zeroclaw agent -a assistant -m “Summarize today’s logs” # single message zeroclaw agent -a assistant -p anthropic –model claude-sonnet-4-20250514 zeroclaw agent -a assistant –peripheral nucleo-f401re:/dev/ttyACM0

Usage: zeroclaw agent [OPTIONS] --agent <AGENT>

Opciones:

  • -a, --agent <AGENT> — Configured agent alias to run as (must match [agents.<alias>]). Required — there is no default agent
  • -m, --message <MESSAGE> — Modo de mensaje único (no entrar en modo interactivo)
  • --session-state-file <SESSION_STATE_FILE> — Cargar y guardar el estado de la sesión interactiva en este archivo JSON
  • -p, --model-provider <MODEL_PROVIDER> — Model provider to use (openrouter, anthropic, openai, openai-codex)
  • --model <MODEL> — Modelo a utilizar
  • -t, --temperature <TEMPERATURE> — Temperature (0.0 - 2.0, defaults to providers.models...temperature)
  • --peripheral <PERIPHERAL> — Adjuntar un periférico (placa:ruta, por ejemplo, nucleo-f401re:/dev/ttyACM0)

zeroclaw gateway

Administra el servidor de puerta de enlace (webhooks, websockets).

Iniciar, reiniciar o inspeccionar la puerta de enlace HTTP/WebSocket que acepta eventos de webhook entrantes y conexiones WebSocket.

Ejemplos: zeroclaw gateway start # iniciar gateway zeroclaw gateway restart # reiniciar gateway zeroclaw gateway get-paircode # mostrar código de emparejamiento

Uso: zeroclaw gateway [COMANDO]

Subcomandos:

  • start — Inicia el servidor de puerta de enlace (predeterminado si no se especifica ningún subcomando)
  • restart — Reiniciar el servidor de puerta de enlace
  • get-paircode — Mostrar o generar el código de emparejamiento sin reiniciar

zeroclaw gateway start

Inicia el servidor de puerta de enlace (webhooks, websockets).

Ejecuta la puerta de enlace HTTP/WebSocket que acepta eventos de webhook entrantes y conexiones WebSocket. La dirección de enlace predeterminada corresponde a los valores definidos en tu archivo de configuración (gateway.host / gateway.port).

Ejemplos: zeroclaw gateway start # usar valores predeterminados de configuración zeroclaw gateway start -p 8080 # escuchar en el puerto 8080 zeroclaw gateway start –host 0.0.0.0 # requiere [gateway].allow_public_bind=true o un túnel zeroclaw gateway start -p 0 # puerto aleatorio disponible

Uso: zeroclaw gateway start [OPTIONS]

Opciones:

  • -p, --port <PORT> — Puerto en el que escuchar (usar 0 para un puerto aleatorio disponible); por defecto usa la configuración gateway.port
  • --host <HOST> — Host al que vincularse; por defecto usa gateway.host de la configuración. Nota: Vincularse a 0.0.0.0 requiere gateway.allow_public_bind = true en la configuración.

zeroclaw gateway reiniciar

Reinicia el servidor de puerta de enlace.

Detiene la puerta de enlace en ejecución, si está presente, y luego inicia una nueva instancia con la configuración actual.

Ejemplos: zeroclaw gateway restart # reiniciar con valores predeterminados de configuración zeroclaw gateway restart -p 8080 # reiniciar en el puerto 8080

Uso: zeroclaw gateway restart [OPTIONS]

Opciones:

  • -p, --port <PORT> — Puerto en el que escuchar (usar 0 para un puerto aleatorio disponible); por defecto usa la configuración gateway.port
  • --host <HOST> — Host al que vincularse; por defecto usa gateway.host de la configuración. Nota: Vincularse a 0.0.0.0 requiere gateway.allow_public_bind = true en la configuración.

zeroclaw gateway obtener-código-de-par

Mostrar o generar el código de emparejamiento del gateway.

Muestra el código de emparejamiento para conectar nuevos clientes sin reiniciar la puerta de enlace. Requiere que la puerta de enlace esté en ejecución.

Con –new, genera un código de emparejamiento nuevo incluso si la puerta de enlace ya estaba emparejada (útil para agregar clientes adicionales).

Examples: zeroclaw gateway get-paircode # show current pairing code zeroclaw gateway get-paircode –new # generate a new pairing code zeroclaw gateway get-paircode –new –port 3001 # target alternate-port gateway

Uso: zeroclaw gateway get-paircode [OPTIONS]

Opciones:

  • --new — Generar un nuevo código de emparejamiento (incluso si ya está emparejado)
  • -p, --port <PORT> — Port of the running gateway to query; defaults to config gateway.port
  • --host <HOST> — Host of the running gateway to query; defaults to config gateway.host

zeroclaw acp

Inicia el servidor ACP (JSON-RPC 2.0 sobre stdio).

Inicia un servidor JSON-RPC 2.0 en stdin/stdout para la integración con IDE y herramientas. Admite la gestión de sesiones y la transmisión de respuestas del agente como notificaciones.

Métodos: initialize, session/new, session/prompt, session/stop.

Ejemplos: zeroclaw acp # iniciar el servidor ACP zeroclaw acp –max-sessions 5 # limitar las sesiones concurrentes

Uso: zeroclaw acp [OPTIONS]

Opciones:

  • --max-sessions <MAX_SESSIONS> — Número máximo de sesiones simultáneas (predeterminado: 10)
  • --session-timeout <SESSION_TIMEOUT> — Tiempo de inactividad de la sesión en segundos (predeterminado: 3600)

zeroclaw daemon

Iniciar el demonio autónomo de ejecución prolongada.

Inicia el entorno de ejecución completo de ZeroClaw: servidor de puerta de enlace, todos los canales configurados (Telegram, Discord, Slack, etc.), monitor de latidos y programador de tareas programadas. Esta es la forma recomendada de ejecutar ZeroClaw en producción o como un asistente siempre activo.

Utiliza ‘zeroclaw service install’ para registrar el demonio como un servicio del sistema operativo (systemd/launchd) para que se inicie automáticamente al arrancar.

Ejemplos: zeroclaw daemon # usar valores predeterminados de configuración zeroclaw daemon -p 9090 # puerta de enlace en el puerto 9090 zeroclaw daemon –host 127.0.0.1 # solo localhost

Uso: zeroclaw daemon [OPTIONS]

Opciones:

  • -p, --port <PORT> — Puerto en el que escuchar (usar 0 para un puerto aleatorio disponible); por defecto usa la configuración gateway.port
  • --host <HOST> — Host al que vincularse; por defecto usa la configuración gateway.host

zeroclaw cron

Configurar y gestionar tareas programadas.

Schedule recurring, one-shot, or interval-based tasks using cron expressions, RFC3339 timestamps with explicit Z or offsets, durations, or fixed intervals.

Cron expressions use the standard 5-field format: ‘min hour day month weekday’. When –tz is omitted, cron schedules use the runtime local timezone. For user-facing schedules, pass –tz with an explicit IANA timezone.

Ejemplos: zeroclaw cron list zeroclaw cron add ‘0 9 * * 1-5’ ‘Buenos días’ –tz America/New_York –agent zeroclaw cron add ‘*/30 * * * ’ ‘Comprobar la salud del sistema’ –agent zeroclaw cron add ‘/5 * * * *’ ‘echo ok’ zeroclaw cron add-at 2025-01-15T14:00:00Z ‘Enviar recordatorio’ –agent zeroclaw cron add-every 60000 ‘Ping heartbeat’ zeroclaw cron once 30m ‘Ejecutar copia de seguridad en 30 minutos’ –agent zeroclaw cron pause TASK_ID zeroclaw cron update TASK_ID –expression ‘0 8 * * *’ –tz Europe/London

Uso: zeroclaw cron <COMANDO>

Subcomandos:

  • list — Listar todas las tareas programadas
  • add — Agregar una nueva tarea programada
  • add-at — Add a one-shot scheduled task at an RFC3339 timestamp with explicit Z or offset
  • add-every — Agregar una tarea programada con un intervalo fijo
  • once — Agrega una tarea diferida de un solo disparo (por ejemplo, “30m”, “2h”, “1d”)
  • remove — Eliminar una tarea programada
  • update — Actualizar una tarea programada
  • pause — Pausar una tarea programada
  • resume — Reanudar una tarea en pausa

zeroclaw cron add

Agrega una nueva tarea programada recurrente.

Uses standard 5-field cron syntax: ‘min hour day month weekday’. When –tz is omitted, cron schedules use the runtime local timezone. For user-facing schedules, pass –tz with an explicit IANA timezone.

Ejemplos: zeroclaw cron add ‘0 9 * * 1-5’ ‘Buenos días’ –tz America/New_York –agent zeroclaw cron add ‘*/30 * * * ’ ‘Comprobar la salud del sistema’ –agent zeroclaw cron add ‘/5 * * * *’ ‘echo ok’

Usage: zeroclaw cron add [OPTIONS] --agent <AGENT_ALIAS> <EXPRESSION> <COMMAND>

Argumentos:

  • <COMMAND> — Command (shell) or prompt (when –prompt) to run

Opciones:

  • --tz <TZ> — Zona horaria IANA opcional (por ejemplo, America/Los_Angeles)
  • --prompt — Treat the argument as an agent prompt instead of a shell command
  • --allowed-tool <ALLOWED_TOOLS> — Restrict agent cron jobs to the specified tool names (repeatable, prompt-only)

The timestamp must include an explicit Z or numeric offset (e.g. 2025-01-15T14:00:00Z or 2025-01-15T09:00:00-05:00).
